🚀 Detalhamento do Desafio: Provisionamento de SQL Database na Microsoft Azure
Este repositório contém o detalhamento dos passos executados para a conclusão do desafio proposto, focado na criação e configuração de recursos de banco de dados na nuvem.

🛠️ Passo a Passo da Execução
1. Acesso ao Portal e Seleção do Recurso
O processo foi iniciado com o acesso ao Portal da Microsoft Azure. Na barra de busca principal, foi localizado e selecionado o serviço Banco de Dados SQL (SQL Database).

2. Configuração do Servidor e Autenticação
Seguindo as diretrizes apresentadas em aula, foi realizada a criação do servidor de banco de dados. Nesta etapa, definiram-se as diretrizes de segurança e os métodos de autenticação do administrador.

    Figura 1: Definição dos parâmetros de autenticação do servidor.

3. Parametrização do Serviço e Vínculo de Assinatura
Na sequência, foi selecionada a assinatura ativa do Azure para o faturamento. O serviço recebeu sua identificação nominal e foi devidamente vinculado ao servidor criado no passo anterior.

    Figura 2: Vinculação de assinatura, nomenclatura e associação do servidor.

4. Dimensionamento e Redundância
Foi realizada a revisão e o ajuste dos recursos computacionais necessários para o projeto, incluindo:

    - Capacidade de processamento (vCPUs/DTUs);

    - Camada de armazenamento;

    - Políticas de desligamento automático;

    - Estratégia de redundância de armazenamento para alta disponibilidade.

    Figura 3: Dimensionamento de hardware e políticas de resiliência.

5. Revisão Final e Precificação
Por fim, a Azure apresentou o resumo consolidado do serviço configurado, exibindo a estimativa de custos e o modelo de precificação adotado antes da criação final do recurso.
